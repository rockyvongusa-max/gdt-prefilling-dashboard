## Target Audience
- Beginner to intermediate Java/Spring Boot developers
- Team members learning the deployment workflow
- Anyone confused about why JAR packaging is required

## Value Proposition
Transforms a dry technical explanation into a visual narrative that answers:
- Why can't we just deploy source code?
- What does a JAR actually contain?
- How does JAR → Docker → running app work?
- What breaks if you skip the build step?

## Core Themes
- Build as transformation/compression metaphor
- App journey from IDE to server
- Failure modes when steps are skipped
- One-click success after proper packaging

## Key Figures & Story Arcs
### Developer/Student
- Role: Learner trying to deploy an app
- Arc: Confused → Understanding → Confident
- Visual trait: Brown hair, glasses, hoodie, holding laptop

### Deployment Mentor (Doraemon-style robot cat)
- Role: Guide who explains the "why"
- Arc: Patient teacher, uses gadgets to show concepts
- Visual trait: Blue robot cat, white belly, 4D pocket, red nose, golden bell

### Error Monster
- Role: Personifies what goes wrong when skipping steps
- Arc: Appears at each failure point
- Visual trait: Red tangled code-text monster, breaks stuff
